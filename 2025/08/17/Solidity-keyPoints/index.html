构造函数

一个合约只能有一个构造函数：初始化的作用

关键字

  • memory：数据存储在内存中，仅在函数执行期间存在。函数调用结束后，存储在  memory  中的数据就会被销毁。就像在一个临时的工作区域处理数据，处理完就清理掉。
  • storage：数据存储在区块链的永久存储中，只要合约存在，这些数据就一直存在。相当于把重要文件存放在一个长期的仓库中。
  • 把函数定义为 view，意味着它只能读取数据不能更改数据。
  • 把函数定义为 pure ，表明这个函数甚至都不访问应用里的数据。
  • public：合约内部可以访问，外部也可以访问。
  • private：仅能被当前合约内部访问，不能被外部合约或账户直接访问。

    注意：
    Solidity 的  private仅限制合约层面的访问权限，但区块链上的所有数据（包括  private  变量）最终都存储在链上的存储槽（storage slot）中，本质上是公开可查的。

    怎么查看？ 1.使用链上数据分析工具（如  web3.js、ethers.js）直接读取合约存储槽数据； 2.分析交易日志或合约存储结构，逆向推导出  private  变量的值。

  • 在调用一个函数之前，可能需要验证一些信息。这个验证步骤就可以写成一个  modifier 修饰器。

数组相关

  • push(x): 动态数组拥有push(x)成员，可以在数组最后添加一个x元素。
  • pop(): 动态数组拥有pop()成员，可以移除数组最后一个元素。

映射

映射的存储位置必须是storage。

导入

在 Solidity 中，当你有多个文件并且想把一个文件导入另一个文件时，可以使用  import  语句。

继承

is 这个关键字说明 A 是 B 的子合约

  • override：子合约重写了父合约中的函数，需要加上override关键字。
  • virtual: 父合约中的函数，如果希望子合约重写，需要加上virtual关键字。

异常

require(检查条件，”异常的描述”)，当检查条件不成立的时候，就会抛出异常。

全局变量

msg.sender是一个全局变量，它代表当前调用（或消息）的发送者的地址。
