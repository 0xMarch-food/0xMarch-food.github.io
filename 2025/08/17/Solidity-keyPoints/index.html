构造函数

一个合约只能有一个构造函数：初始化的作用

关键字

  • memory：数据存储在内存中，仅在函数执行期间存在。函数调用结束后，存储在  memory  中的数据就会被销毁。就像在一个临时的工作区域处理数据，处理完就清理掉。
  • storage：数据存储在区块链的永久存储中，只要合约存在，这些数据就一直存在。相当于把重要文件存放在一个长期的仓库中。
  • 把函数定义为  view，意味着它只能读取数据不能更改数据。
  • 把函数定义为 pure ，表明这个函数甚至都不访问应用里的数据。
  • public：合约内部可以访问，外部也可以访问。
  • private：仅能被当前合约内部访问，不能被外部合约或账户直接访问。

    注意：
    Solidity 的  private仅限制合约层面的访问权限，但区块链上的所有数据（包括  private  变量）最终都存储在链上的存储槽（storage slot）中，本质上是公开可查的。

    怎么查看？ 1.使用链上数据分析工具（如  web3.js、ethers.js）直接读取合约存储槽数据； 2.分析交易日志或合约存储结构，逆向推导出  private  变量的值。

  • 在调用一个函数之前，可能需要验证一些信息。这个验证步骤就可以写成一个  modifier 修饰器。

数组

数组用T[k]的格式声明，其中T是元素的类型，k是长度。k 也可以不写。

对于memory修饰的动态数组，可以用new操作符来创建，但是必须声明长度，并且声明后长度不能改变。例如 bytes[] memory arr = new bytes[](2)。

  • push(x):  动态数组拥有push(x)成员，可以在数组最后添加一个x元素。
  • pop():  动态数组拥有pop()成员，可以移除数组最后一个元素。

映射

映射的存储位置必须是storage。

modifier 修饰器

在调用函数之前，可能需要验证一些信息。这个验证步骤就可以写成一个modifier。
当你定义好这个modifier并且应用到函数上之后，每次调用这个函数，就会先执行modifier里的验证规则，只有验证通过了，函数的主体部分才会执行。

导入

在 Solidity 中，当你有多个文件并且想把一个文件导入另一个文件时，可以使用  import  语句。

接口

  1. Functions in interfaces must be declared external.
  2. Functions in interfaces cannot have modifiers.
  3. 接口不仅能写函数，还能写状态变量。

继承

is 这个关键字说明 A 是 B 的子合约

  • override：子合约重写了父合约中的函数，需要加上override关键字。
  • virtual: 父合约中的函数，如果希望子合约重写，需要加上virtual关键字。

异常

require(检查条件，”异常的描述”)，当检查条件不成立的时候，就会抛出异常。

全局变量

  • msg.sender是当前调用者的地址。
  • address(this)是当前合约的地址。
  • msg.value ：与消息一起发送的 wei 数量
  • tx.origin ：交易的发送者地址（完整调用链)

转换

类型 含义 占用空间（字节） 取值范围
address 以太坊地址（如你的钱包地址） 20 字节 0x000…000 ~ 0xfff…fff（共 20 字节）
uint160 无符号 160 位整数 20 字节（160 位） 0 ~ 2¹⁶⁰ - 1
uint256 无符号 256 位整数（常用） 32 字节（256 位） 0 ~ 2²⁵⁶ - 1

Solidity 不允许 “跨尺寸直接转换”，防止数据丢失：比如uint256(uint160(msg.sender))，因为  address不能直接转  uint256，但  address  和  uint160  尺寸完全一致，转换时就不会丢失任何数据。
