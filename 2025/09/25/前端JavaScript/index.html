JavaScript 异步编程

异步（Asynchronous, async）是与同步（Synchronous, sync）相对的概念。
简单来理解就是：同步按你的代码顺序执行，异步不按照代码顺序执行，异步的执行效率更高。![[../区块链安全/Ether.js/photo/Pasted image 20250126120137.png]]
Js单线程，同一个时间只能做一件事 ——-> 任务需要排队，如果JS执行的时间过长，这样就会造成页面的渲染不连贯，导致页面渲染加载阻塞 ——-> 为解决这个问题，HTML5提出Web Worker标准，允许JS脚本创建多个线程，于是，JS出现了同步与异步

异步一般有一线三种类型:

  • 普通事件，如click、resize
  • 资源加载，如load、error
  • 定时器，如setInterval、setTimeout

Promise类

Promise 是一个”承诺”，表示将来某个时间点会返回一个结果（可能是成功的结果，也可能是失败的原因）。

then() 方法用于指定 Promise 状态变为 fulfilled 或 rejected 时的回调函数。
catch() 方法专门用于处理 Promise 被拒绝的情况。
finally() 方法无论 Promise 最终状态如何都会执行。

Promise的链式调用
Promise 的一个强大特性是可以链式调用多个异步操作

Promise的静态方法
Promise.all()等待所有 Promise 完成，或任意一个 Promise 失败。
Promise.race()返回最先完成（无论成功或失败）的 Promise 的结果。

async/await

  • 在async/await出现之前，JavaScript主要使用回调函数处理异步操作，但这会容易导致”回调地狱”。
  • 后来引入了Promise类来解决这个问题，虽然改善了，但是then()链式调用不够直观。
  • ES2017 引入了 async/await，它建立在 Promise 之上，让异步代码看起来像同步代码一样。
    在函数声明前添加async关键字，表示该函数是异步的;await只能在async函数内部使用。await会暂停async函数的执行，等待Promise完成。

如果需要并行执行多个异步操作，可以使用Promise.all。
