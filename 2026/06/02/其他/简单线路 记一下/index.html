基本需求：
1.掌握一门基本的现代web开发语言(java/ts/golang/python/rust)，语言层面不做限制
2.了解基本的web开发框架(Spring Boot/Express/Gin/FastAPI/Axum等)，能搭建一个基本的web服务
3.了解一种数据库的基本使用(PostgreSQL, MySQL)，会sql语言和数据库操作框架
4.了解一种缓存数据库的基本使用(Redis)，各种同步方式和数据结构的使用
5.了解基本的HTTP协议和RESTful API设计原则，能设计和实现一个简单的CRUD API
6.了解基本的版本控制工具(Git)，能使用Git进行代码管理和协作，考核项目要提交到github上
7.了解基本的Linux基本命令，能在Linux环境下进行开发和部署，建议用WSL
8.了解基本的容器化技术(Docker)，能使用Docker进行应用容器化和部署
9.建议写个前端（React/Vue），但不强制，后端接口要能被前端调用

项目考核要求：
1.做一个web项目，技术栈不限，题目会在后续公布，但是肯定会有agent项目
2.鼓励使用AI进行SDD+TDD的开发，建议用codex或者claude code，架构和业务逻辑要自己想好
3.托管到github上，提交PR，要求代码规范，README清晰，能用docker-compose部署运行
4.项目的架构业务逻辑，以及出现的所有技术栈都在考察范围。主要考察架构设计和业务思维，具体实现交给AI没关系，但要能解释清楚为什么这么设计，为什么选这个技术栈，为什么这么实现。不管你是拿开源项目改的还是自己写的，只要能解释清楚为什么这么设计，这个项目就是你的了。
5.如果是agent项目，除了基本的llm调用，rag，mcp，skill，tool calling之外，可以做一些权限校验，流式返回，workflow编排，multi-agent协作等进阶功能的实现，这个是加分项。
